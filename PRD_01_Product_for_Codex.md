# PRD 01 — Product PRD для Codex

## 1. Название проекта
AI Cat Notes & Chat

## 2. Цель продукта
Создать mobile-first приложение для **iOS, Android и браузера**, в котором пользователь может:
- быстро сохранять идеи в виде текста или голосовой заметки;
- получать от ИИ аккуратную суммаризацию и переработанный вариант идеи;
- общаться с ИИ в режиме чата текстом и голосом;
- видеть дружелюбного пиксельного котика, который сопровождает навигацию и позже будет анимирован глубже.

MVP должен работать **локально на устройстве**, без обязательной регистрации. Архитектура должна быть подготовлена к будущей синхронизации между устройствами.

## 3. Платформы
- iOS
- Android
- Web (адаптивный браузерный клиент)

## 4. Продуктовые принципы
- Mobile first
- Быстрый первый запуск
- Минимум трения до первого полезного действия
- Локальное хранение в MVP
- Подготовка к облачной синхронизации
- Приятный визуальный стиль в кремовых тонах
- Простая, понятная навигация
- Двуязычность: русский и английский

## 5. Основные сценарии пользователя

### Сценарий A — ИИ заметки
1. Пользователь открывает вкладку «Заметки».
2. Нажимает «Новая заметка».
3. Вводит текст или нажимает запись голоса.
4. После завершения ввода нажимает «Обработать».
5. Приложение отправляет содержимое в ИИ.
6. ИИ возвращает:
   - короткое резюме;
   - улучшенный и более понятный вариант идеи;
   - необязательные структурные блоки: цель, суть, следующие шаги.
7. На экране по умолчанию показывается **обработанный текст**.
8. Кнопка **«Оригинальный текст»** раскрывает выпадающий блок.
9. При нажатии на этот пункт текущий отображаемый текст заменяется на оригинальный.
10. Пользователь может снова вернуться к обработанной версии.

### Сценарий B — Режим общения с ИИ
1. Пользователь открывает вкладку «Чат».
2. Создаёт новый чат или открывает существующий.
3. Пишет сообщение или надиктовывает голосом.
4. Речь преобразуется в текст.
5. Текст уходит в ИИ.
6. ИИ отвечает текстом.
7. Сообщения сохраняются в истории локально.
8. Пользователь может:
   - создать новый чат;
   - переименовать чат;
   - удалить чат;
   - открыть старый чат и продолжить диалог.

## 6. Функциональные требования MVP

### 6.1 Навигация
Нижняя навигация из 2 основных вкладок:
- Заметки
- Чат

Дополнительно:
- экран настроек или sheet/side panel;
- языковой переключатель;
- заготовка для будущих настроек модели и анимаций.

### 6.2 ИИ заметки
Функции:
- создание заметки вручную;
- создание заметки через голосовой ввод;
- отправка текста в ИИ;
- получение структурированного результата;
- сохранение оригинального текста;
- переключение между обработанным и оригинальным текстом;
- редактирование названия заметки;
- удаление заметки;
- локальный список заметок;
- сортировка по последнему изменению.

Поля заметки:
- `id`
- `title`
- `originalText`
- `processedText`
- `summary`
- `language`
- `createdAt`
- `updatedAt`
- `sourceType` (`text` | `voice`)
- `status` (`draft` | `processed` | `error`)

### 6.3 Режим чата
Функции:
- список чатов;
- создание нового чата;
- переименование чата;
- удаление чата;
- история сообщений внутри чата;
- текстовый ввод;
- голосовой ввод с преобразованием в текст;
- текстовый ответ ИИ;
- отображение состояния отправки и ошибки.

Поля чата:
- `id`
- `title`
- `createdAt`
- `updatedAt`
- `lastMessagePreview`
- `messageCount`

Поля сообщения:
- `id`
- `chatId`
- `role` (`user` | `assistant` | `system`)
- `content`
- `createdAt`
- `sourceType` (`text` | `voice`)
- `model`
- `status`

### 6.4 Голосовой ввод
MVP:
- запись голоса;
- распознавание речи в текст;
- просмотр текста перед отправкой;
- ручное редактирование распознанного текста.

Важно:
- ИИ пока отвечает **только текстом**;
- голосовой ответ от ИИ не нужен в MVP.

### 6.5 Локализация
Языки:
- Русский
- English

Переводу подлежит:
- весь интерфейс;
- системные состояния;
- кнопки;
- заголовки;
- пустые состояния;
- сообщения ошибок.

Контент, который вводит пользователь, не переводится автоматически.

### 6.6 Пиксельный котик
На этапе MVP нужен **placeholder-персонаж**:
- отображается в шапке или как закреплённый UI-элемент;
- имеет состояния-заглушки для будущих анимаций;
- при переходе между вкладками страница уезжает влево/вправо;
- кот меняет состояние в зависимости от направления навигации.

На текущем этапе полноценная анимация не обязательна.

## 7. Нефункциональные требования
- Интерфейс должен быстро загружаться на мобильных устройствах.
- Приложение должно корректно работать в браузере и на телефоне.
- Данные в MVP должны сохраняться локально между перезапусками.
- UI должен быть отзывчивым при плохом интернете.
- При недоступности ИИ пользователь не должен терять локальные данные.
- Все опасные действия должны иметь подтверждение или undo-паттерн.

## 8. UX и визуальный стиль

### Цветовая палитра
Основной стиль:
- кремовые, тёплые, мягкие оттенки;
- аккуратные тени;
- минимализм без перегруза;
- карточный интерфейс.

### UI-направление
- крупные touch-target элементы;
- один главный CTA на экран;
- нижняя навигация;
- плавные переходы;
- мягкие состояния hover/focus/pressed;
- дружелюбные пустые состояния.

### Экран заметок
- список заметок карточками;
- FAB / кнопка создания новой заметки;
- карточка заметки показывает заголовок, краткое summary, дату изменения;
- в деталке заметки сверху обработанный вариант;
- под ним кнопка/аккордеон «Оригинальный текст».

### Экран чатов
- список чатов;
- кнопка нового чата;
- long press / меню для rename/delete;
- внутри чата классическая лента сообщений;
- поле ввода + кнопка микрофона + кнопка отправки.

## 9. Ограничения и правила безопасности
- API-ключ OpenRouter **не должен храниться в клиентском приложении**.
- Все запросы к OpenRouter должны идти через серверный proxy/BFF.
- Локальные данные должны быть доступны без логина в MVP.
- Запросы к ИИ не должны блокировать UI.
- Нужно предусмотреть лимиты и обработку ошибок API.

## 10. Информационная архитектура
- `/notes`
- `/notes/:id`
- `/chat`
- `/chat/:id`
- `/settings`
- `/about` (опционально)

## 11. Состояния пустого экрана

### Notes empty state
- Заголовок: «Сохраните первую идею»
- Подзаголовок: «Напишите мысль или надиктуйте её, а ИИ поможет оформить её понятнее»

### Chat empty state
- Заголовок: «Начните новый разговор»
- Подзаголовок: «Напишите сообщение или используйте голосовой ввод»

## 12. Ошибки и fallback-сценарии
- Если не удалось распознать голос — показать текст ошибки и кнопку повторить.
- Если ИИ не ответил — сохранить исходный текст и дать retry.
- Если обработка заметки упала — заметка остаётся в статусе draft/error.
- Если чат не отправился — сообщение помечается как failed и может быть переотправлено.

## 13. Будущие функции после MVP
- синхронизация между устройствами;
- авторизация;
- облачное хранение заметок и чатов;
- настройка моделей;
- выбор системного промпта;
- голосовые ответы ИИ;
- полноценные анимации котика;
- экспорт заметок;
- теги и папки;
- поиск по заметкам и чатам.

## 14. Критерии готовности MVP
MVP считается готовым, если:
1. Приложение открывается на iOS, Android и в браузере.
2. Пользователь может создать заметку текстом и голосом.
3. ИИ возвращает summary и улучшенный вариант текста.
4. Оригинальный текст можно открыть и подменить им текущий.
5. Пользователь может создать несколько чатов.
6. Чаты можно переименовывать и удалять.
7. История чатов сохраняется локально.
8. Интерфейс доступен на русском и английском.
9. Есть базовая заготовка пиксельного котика и переходов между вкладками.
10. OpenRouter подключён через proxy, без утечки ключа на клиент.
